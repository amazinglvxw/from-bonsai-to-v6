After the Cyber Bonsai audit, we did what any self-respecting engineer would do when confronted with failure: we read papers. A lot of papers.

## The Research Sprint

From March 20 to March 28, we analyzed 93 academic papers, open-source projects, and developer blogs. Each was evaluated through dual-AI analysis — Claude for depth, Gemini for contrarian perspective.

## The Five Themes That Emerged

Across 93 papers, five themes kept surfacing:

### 1. Harness > Model (6 independent validations)

From Sydney's SWE-agent to Anthropic's own guidelines, from MetaClaw to Training-Free GRPO: the system around the model matters more than the model itself. Our 800-line rule file was the anti-harness — it tried to make the model smarter instead of making the system more reliable.

### 2. Simple Systems That Run > Complex Systems That Don't

yoyo-evolve grew from 200 to 31,000 lines with a simple JSONL log and cargo test. OpenSpace built a full self-evolving skill engine. Both had one thing in common: **real execution logs feeding real feedback loops**.

### 3. Direct Sharing Hurts — Distill First

MemCollab proved empirically that sharing raw memory between agents *degrades* performance. You must distill through contrastive analysis first. Our A2A protocol was naive.

### 4. Cheap Verifiers Beat LLM Self-Evaluation

ACG Survey's Guideline 7.4, NLAH's deterministic adapters, yoyo-evolve's cargo test — every source agreed: use code to verify, not LLMs.

### 5. Context Durability Is the Next Bottleneck

Philipp Schmid at Hugging Face named it. Our 800-line rule file was the anti-pattern — causing the very "context fatigue" it was designed to prevent.

## What Changed

These 93 papers didn't give us a new architecture. They gave us **permission to simplify**. The answer was never in the next paper — it was in the execution log we never wrote to.
